# 项目结构
## 一个项目要有一个合理的文件夹结构

~~~
root/
├── 📂 backend/                 # 后端工程 (Python/Go/Node.js)
└── 📂 frontend/                # 前端工程 (React + Vite)
    ├── 📂 public/              # 静态资源 (打包时不压缩，如favicon, 标题图片)
    ├── 📂 node_modules/        # 项目依赖包
    ├── 📂 src/                 # 源代码核心
    │   ├── 📂 api/             # API 接口请求 (Axios 封装, Dify/RAG 接口)
    │   ├── 📂 assets/          # 静态资源 (样式表, 图片, 字体)
    │   ├── 📂 components/      # 公共 UI 组件 (按钮、弹窗等原子组件)
    │   ├── 📂 hooks/           # 自定义 Hooks (React 的逻辑复用，等同于 Vue 的 hooks/composables)
    │   ├── 📂 layout/          # 页面布局 (如侧边栏、顶部导航 Navbar)
    │   ├── 📂 router/          # 路由配置 (基于 react-router-dom)
    │   │   └── 📄 index.jsx    # 路由表定义
    │   ├── 📂 store/           # 状态管理 (MobX)
    │   │   ├── 📄 navbar.js    # 存放全局状态变量和函数，用 @observable 表示该变量是响应式的，在类组件中通过 inject 注入后，用 this.props.navbar 访问
    │   │   └── 📄 resume.js    # 简历相关的全局状态,这里shi
    │   ├── 📂 utils/           # 工具函数 (格式化、校验、本地存储)
    │   ├── 📂 views/           # 页面级组件 (有时在 React 中也常命名为 pages)
    │   │   ├── 📄 HomeView.jsx   # 首页组件 (React 中使用 .jsx 或 .tsx 扩展名)
    │   │   └── 📄 CourseView.jsx # AI 互动课页面
    │   ├── 📄 App.jsx          # 根组件 (视图入口，包含路由守卫或全局 Provider)
    │   └── 📄 main.jsx         # 入口文件 (通过 ReactDOM.createRoot 挂载到 index.html，并包裹 MobX 的 Provider)
    ├── 📄 .env                 # 环境配置文件 (API 地址等，Vite 中以 VITE_ 开头)
    ├── 📄 index.html           # 页面入口 HTML (内部引入 src/main.jsx)
    ├── 📄 jsconfig.json        # 路径别名配置 (让 @ 映射到 src)
    ├── 📄 package.json         # 项目依赖与脚本说明 (包含 react, react-dom, mobx, mobx-react-lite/mobx-react 等)
    └── 📄 vite.config.js       # Vite 配置文件 (配置 React 插件及打包规则)
~~~

这里view最好不要含有任何逻辑相关的内容,只包含将视图放到 < template > 里的逻辑写上
